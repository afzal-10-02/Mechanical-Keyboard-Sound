# Contributing Guide

Thank you for your interest in contributing to this project!
This guide will help you understand the contribution workflow and how to submit your changes.

1. Fork the Repository

Click the **Fork** button on the top right of the repository to create your own copy.

2. Clone Your Fork

Clone the forked repository to your local machine.

git clone https://github.com/YOUR-USERNAME/REPOSITORY-NAME.git

Navigate into the project directory.
cd REPOSITORY-NAME

3. Create a New Branch

Always create a new branch before making changes.
git checkout -b feature/your-feature-name

Example:
git checkout -b docs/add-contribution-guide

4. Make Your Changes

Implement the required changes or fixes in your local environment.

Make sure to:

* Follow the project's coding style
* Keep commits small and meaningful
* Test your changes before committing

5. Commit Your Changes

Add and commit your changes with a clear commit message.

git add .
git commit -m "docs: add contribution guide"

6. Push Your Changes

Push the branch to your forked repository.
git push origin feature/your-feature-name

7. Create a Pull Request

1. Go to your forked repository on GitHub
2. Click **Compare & Pull Request**
3. Add a clear title and description
4. Submit the Pull Request

Contribution Tips

* Keep pull requests focused on a single change
* Write meaningful commit messages
* Follow the existing project structure
* Check if an issue already exists before starting work

Need Help?

If you have any questions, feel free to open an issue or ask in the project discussions.

We appreciate your contributions to the project!
